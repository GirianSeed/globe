基本グラフィックスライブラリ libgp のごく基本的な使い方サンプル

< サンプルの解説 >

	基本グラフィックスライブラリ libgp の、もっとも基本的な使い方
	を紹介する導入用サンプルです。

< ファイル >

	main.c		メインプログラムソース
			画面の切替とクリア、コントローラ入力
	oneprim.c	一つのスプライトパケットだけを描画する
	prims.c		チェインを使用して、３つのパケットを描画する
	defines.h	共通ヘッダファイル
	Makefile	メイクファイル

< 起動方法 >

        % make run

< コントローラの操作方法 >

	SELECTボタン		描画ルーチンの切替
	↑↓→←(方向キー)	(prims.c のときのみ) 操作対象の切り替え
	○△×□ボタン		パケット位置操作
	
< サンプルの説明 >

       oneprim.c	
	　　スプライトパケットを一つだけ描画します。
    	    このサンプルは一つのパケットを単独でDMA転送して描画し、
	　　チェインは使用しません。

    	    oneprim_init()  スプライトパケットの生成と内容の設定を
			　　おこないます。
    	    oneprim_draw()  パケット内容の更新と、実際の描画 (パケットの
			    単独転送)をおこないます。
    	    oneprim_exit()  パケットで使用したメモリ領域を破棄します。


       prims.c	
    	    複数のパケットを続けて転送する場合は、このサンプルのように
	    チェインを使用します。
    	    Z 値の異なる3つのパケットを一つのチェインにつなげて転送します。
    
    	    prims_init()  3つのパケットの生成と内容の設定をおこないます。
    			  チェインの生成と初期化をおこない、パケットを
    			  チェインに登録します。

    	    prims_draw()  パケット内容の更新と、実際の描画 (チェインの
	        	    転送) をおこないます。

    	    prims_exit()  パケットとチェインで使用したメモリ領域を
			  破棄します。


       main.c
    	    pad_init() コントローラ入力のための初期化処理をおこないます。
    	    pad_read() コントローラ入力を読み込みます。

    	    db_init() GS のダブルバッファリング (描画/表示用) の初期設定を
		　　　します。
    	    db_swap() GS のダブルバッファリング (描画/表示用) を切り替えます。

    	    実際の gp による描画は while ループ中の 
    		  mode_list[mode].draw_func(paddata, padtrig);
    	    から呼び出しています。

       defines.h
    	    各ファイルで共通に使用する定数やインライン関数を宣言しています。
    	    convert_x(), convert_y() 頂点座標をGS プリミティブ座標での値に
	　　変換します。

    	さらに詳細はプログラム内のコメントと、ライブラリマニュアル（概要
	およびリファレンス）を参照してください。
    
    	テクスチャつきの描画や透視変換を方法については、ほかのサンプル 
	(zsortなど) を参照してください。

< 注意事項 >

	読みやすさに重点をおいたサンプルのため、動作効率で最適なものでは
	ありません。たとえば、index 取得関数は type を引数にして汎用の 
	index 関数を使用していますが、動作効率の点からは、type 別の
	専用関数を使用すべきです。
